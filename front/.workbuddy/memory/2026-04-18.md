## (home) layout 适配项目技术栈

**任务**：用户复用了 Umi.js + Ant Design 旧项目的 layout，将其改造为本项目技术栈。

**改动文件**：
- `src/app/(home)/layout.tsx` — 完整重写
- `src/app/(home)/layout.module.less` — 样式适配
- `src/middleware.ts` — 顺带修复 ADMIN_ROUTES 空数组隐式 any[] 类型

**替换内容**：

| 旧依赖（Umi/Antd） | 新实现（Next.js/Lucide）|
|:---|:---|
| `import { Link, Outlet } from "umi"` | `import Link from 'next/link'` + `children` prop |
| `import { FloatButton } from "antd"` | 手写 FAB 浮动按钮组（CSS + state 控制展开收起） |
| `@ant-design/icons` | `lucide-react`（Settings / X / ArrowUp / Sun / Moon / Monitor） |
| `import language from "@/language/language"` | 中文硬编码（当前项目无国际化） |
| `import logo from "../assets/logo-white.svg"` | Lucide `PenSquare` 图标 |
| 硬编码颜色（#3498db 等） | CSS 变量（var(--color-primary) 等），支持暗色主题 |
| 内联 `debounce`（含 `Function`/`any` 类型） | 复用 `@/lib/utils` 的类型安全 `debounce` |

**保留的交互逻辑**：整屏滚动、滚动感知导航栏动画

## /home 页面空 src 错误修复

**问题定位**：`src/app/(home)/home/page.tsx` 是旧 OSS Manager 遗留文件，渲染 `<Player videoSrc="" />`（默认值空字符串），导致 `<video src="">` 被浏览器渲染并报警告。

**根因**：`videoPlayer/index.tsx` 的 `videoSrc` 默认值为 `""`（空字符串），两个 `<video>` 元素无条件渲染了 `src={videoSrc}`。

**修复**：
1. `(home)/home/page.tsx` → 改为 `redirect('/')`，不再使用有问题的 Player 组件
2. `videoPlayer/index.tsx` → `videoSrc` 默认值改为 `undefined`，两个 `<video>` 元素改为条件渲染 `{videoSrc && <video src={videoSrc} .../>}`

**构建结果**：✅ 11 个页面，0 错误
