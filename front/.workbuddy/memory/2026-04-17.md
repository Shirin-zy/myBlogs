## 登录页类型错误修复

**任务**：用户对登录页进行手动调整后，`page.tsx` 和 `AnimatedCharacters.tsx` 存在 TypeScript 类型问题，进行修复。

**修复内容**：

| 文件 | 位置 | 问题 | 修复方案 |
|:---|:---|:---|:---|
| `page.tsx` | `onSubmit` 函数 | 参数 `e` 隐式 `any` | 改为 `React.FormEvent<HTMLFormElement>` |
| `AnimatedCharacters.tsx` | `Pupil` 组件定义 | props 无类型 / `forceLookX/Y` 隐式 `any` | 新增 `PupilProps` 接口 |
| `AnimatedCharacters.tsx` | `EyeBall` 组件定义 | props 无类型 / `forceLookX/Y` 隐式 `any` | 新增 `EyeBallProps` 接口 |
| `AnimatedCharacters.tsx` | `AnimatedCharacters` 主组件 | props 无类型 | 新增 `AnimatedCharactersProps` 接口 |
| `AnimatedCharacters.tsx` | `useRef(null)` ×4 | 推断为 `never` | 改为 `useRef<HTMLDivElement>(null)` |
| `AnimatedCharacters.tsx` | `mousemove` 事件回调 ×3 | 参数 `e` 隐式 `any` | 改为 `MouseEvent` |
| `AnimatedCharacters.tsx` | `calculatePosition(ref)` | 参数 `ref` 隐式 `any` | 改为 `React.RefObject<HTMLDivElement \| null>` |

**结果**：`tsc --noEmit` 0 错误 ✅

## Less 集成 + 登录页样式迁移

**任务**：将登录页 CSS 改写为 Less，并集成到 Next.js 15 项目中。

**操作步骤**：
1. 安装 `less` + `less-loader`（devDependencies）
2. 修改 `next.config.js`：webpack 钩子中克隆内置 CSS Modules 规则，追加 `less-loader`，匹配 `/\.module\.less$/`
3. 新增 `src/types/less.d.ts`：添加 `.module.less` 类型声明
4. 创建 `src/app/(auth)/login/login.module.less`：CSS 改写为 Less，使用变量（@color-*）、嵌套选择器、`.mixin()` 复用 transition/flex-center/absolute-fill 逻辑
5. 更新 `page.tsx`：`import styles from './login.module.less'`，所有 `className="xxx"` 替换为 `className={styles.xxx}`

**踩坑**：
- Next.js 15 不内置 Less 支持，单独插入 less-loader 规则会报 "You may need an additional loader"（缺 css-loader）
- 正确做法：找到 Next.js 已有的 CSS Modules oneOf 规则，克隆 + 末尾加 less-loader，共享 css-loader/MiniCssExtractPlugin

**构建结果**：✅ `next build` 通过，11 个页面生成，0 错误
