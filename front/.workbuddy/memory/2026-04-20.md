## 登录接口封装与登录逻辑接入

**涉及文件**：
- `.env.development` / `.env.production`：环境变量区分
- `src/services/auth-service.ts`：新建登录 API 封装
- `src/stores/auth-store.ts`：接入真实接口 + Token 存 Cookie
- `src/app/(auth)/login/page.tsx`：onSubmit 接入 store.login + 成功跳转
- `src/middleware.ts`：启用后台路由保护

**关键决策**：
- 后端登录响应格式为 `{ status, message, data: { token, user_info } }`，与项目通用 `{ code, message, data }` 不同，单独用原生 fetch 封装 `loginApi`，不走 `http.ts`
- Token 使用 `js-cookie` 存入 `blog-session` cookie（客户端可读），供 middleware 路由守卫读取
- `middleware.ts` ADMIN_ROUTES 启用：`['/dashboard', '/posts', '/audit', '/settings']`
- 环境区分：开发 `http://127.0.0.1:5173/api`，生产 `https://api.yourdomain.com/api`
