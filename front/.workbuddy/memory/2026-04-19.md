# 2026-04-19 工作日志

## (home) layout 导航白线修复 + 滚动条隐藏

**问题1**：回到顶部时导航栏短暂出现白线。
**根因**：`.top` 状态使用了 `transform: translateY(-20px)`，在过渡期间 `border-bottom` + 毛玻璃背景没有被同步过渡，边缘区域短暂露出白色。

**修复**：
- 将 `border-bottom` 改为 `::after` 伪元素实现，通过 `opacity: 0/1` 控制显隐，平滑过渡不产生硬边
- `.top` 状态改为 `opacity: 0` + `pointer-events: none`（而非 `translateY(-20px)`），彻底避免边界残留
- 移除 `.scrolled` 中的 `border-bottom: 1px solid ...` 直接写法

**问题2**：`/home` 页面滚动条可见。
**修复**：
- `globals.css` 新增 `html.hide-scrollbar` + `html.hide-scrollbar body` + `html.hide-scrollbar ::-webkit-scrollbar { display: none }` 全平台隐藏规则
- `(home)/layout.tsx` 在 `useEffect` 挂载时为 `document.documentElement` 添加 `hide-scrollbar` class，卸载时移除，不影响其他路由组

**构建结果**：✅ 11 个页面，0 错误

## 主题切换作用域隔离 v2（修复 dashboard 不生效问题）

**问题**：v1 方案使用双层 ThemeProvider + `forcedTheme="light"` 导致 dashboard 主题不生效。
**根因**：next-themes 的每个 ThemeProvider 都直接操作同一个 `document.documentElement.classList`，且不共享 React context；`forcedTheme` 在根层持续锁定 html class 为 `light`，覆盖了 dashboard 写入的 `dark`。

**最终方案 v5**：CSS 变量硬编码，彻底切断与 html.dark 的关联

彻底放弃与 next-themes 的 `html.dark` class 机制对抗，改为让 home 路由组使用**独立 CSS 变量域**：

- `layout.module.less` 新增 `.homeRoot` 类：定义所有 `--color-*` 变量为亮色固定值
- `layout.tsx` 的根 div 同时应用 `cn(styles.homeRoot, styles.layout)`
- CSS 层叠规则：`.homeRoot` 的变量定义在 `.layout` 内部，作用域覆盖其下所有子元素
- **核心原理**：CSS custom property 的继承机制——`.homeRoot` 上的变量向下传递给所有子元素，无论 `html` 是否有 `dark` class，都被 `.homeRoot` 覆盖
- Dashboard 路由组不受影响：dashboard layout 不使用 `styles.homeRoot`，继续正常响应 `html.dark` class

**构建结果**：✅ 11 个页面，0 错误

## Hydration Mismatch 错误修复

**问题**：React 服务端与客户端渲染不一致，报错 `A tree hydrated but some attributes of the server rendered HTML didn't match the client properties`

**根因**：
1. `descCard/index.tsx`：在 JSX 中直接调用 `Math.random()` 生成随机图标
2. `dashboard-overview.tsx`：模块级别使用 `Date.now()` 生成动态时间戳

**修复方案一**（采用）：
- `descCard`：使用 `useState` + `useEffect` 在客户端渲染后生成随机数据，`beltItems` 初始为空数组
- `dashboard-overview`：使用固定时间戳替代 `Date.now()`

**构建结果**：✅ 0 错误
