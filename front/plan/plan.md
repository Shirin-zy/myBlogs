## 📝 项目介绍

本项目是一个现代化的 **个人博客系统**，旨在为博主提供一个简洁、高效的内容创作与管理平台，同时为读者提供流畅、优雅的阅读体验。系统分为 **普通用户访问端 (Public Site)** 和 **管理后台 (Admin Dashboard)** 两大部分。

- **普通用户**：可自由浏览所有文章、分类、标签，并进行搜索。
- **管理员**：登录后可查看网站运行数据概览，管理文章创作、评论审核及系统配置。

---

## ✨ 功能特性

### 核心功能

- **文章管理**：支持 Markdown 写作、实时预览、封面图上传及存稿/发布切换。
- **数据概览**：管理员可实时查看文章总数、今日访问量、评论数及用户地域分布。
- **互动评论**：支持用户留言评论，管理员可进行审核、回复或删除。
- **分类与标签**：灵活的文章分类体系，支持多级分类与标签聚合。
- **全局搜索**：基于标题、摘要及内容的全文模糊搜索。
- **响应式设计**：完美适配 PC、平板及手机移动端阅读。

## 🛠️ 模块设计

### 3.1 普通用户访问端
- **3.1.1 首页与展示**
  - 文章列表：分页/瀑布流展示文章标题、摘要、封面及发布日期。
  - 侧边栏：博主简介、热门标签、最新评论、分类统计。
- **3.1.2 文章阅读**
  - Markdown 渲染：支持代码高亮、数学公式、流程图。
  - 目录导航 (TOC)：根据文章标题自动生成侧边目录。
  - 相关推荐：基于标签推荐相似文章。
- **3.1.3 资源索引**
  - 搜索：支持标题和内容的实时模糊匹配。
  - 归档页：按年份/月份对文章进行时间轴排列。

### 3.2 管理后台模块
- **3.2.1 数据工作台 (Admin Only)**
  - **核心指标**：展示文章总数、分类总数、评论总数。
  - **访问统计**：今日 PV/UV、过去 7 天访问趋势图、热门文章排行。
- **3.2.2 内容创作**
  - 文章编辑器：支持 Markdown/富文本切换，自动保存草稿。
  - 媒体管理：图片拖拽上传、剪切板粘贴上传。
- **3.2.3 评论与互动**
  - 待审核列表：集中处理待审核的评论，防止垃圾广告。
  - 快速回复：在后台直接对用户评论进行回复。
- **3.2.4 系统配置**
  - 站点信息：网站标题、Logo、备案号配置。
  - 个人资料：博主昵称、头像、社交媒体链接修改。

## 🛡️ 非功能设计

- **🚀 性能优化**
  - **服务端渲染 (SSR)**：提升 SEO 效果及首屏加载速度。
  - **图片优化**：自动生成 WebP 格式，根据设备分辨率加载不同尺寸图。
  - **静态导出 (SSG)**：对不常变动的页面进行预渲染。
- **🔒 安全保障**
  - **管理端防护**：后台登录入口隐藏，支持二次验证 (2FA)。
  - **内容安全**：对用户评论进行严格的 XSS 过滤及敏感词屏蔽。
- **🎨 用户体验**
  - **主题切换**：支持跟随系统或手动切换亮色/暗色模式。
  - **加载反馈**：骨架屏 (Skeleton) 渲染，平滑的页面转场动画。

## 🛠️ 开发规范

为了保证项目的高质量交付与可持续维护，所有开发者需严格遵守以下规范：

### 1. 🚀 性能优化规范
- **渲染优化**：
  - 文章列表必须使用 Next.js 的分页或无限滚动加载。
  - 复杂组件需合理使用 `React.memo` 等 Hooks 防止不必要的重复渲染。
- **资源管理**：
  - 文章封面必须实现懒加载，并使用 `next/image` 进行自动优化。
  - 搜索操作必须应用防抖 (Debounce)。

### 2. 🛡️ 安全编码规范
- **输入校验**：
  - 评论内容必须进行 XSS 过滤，管理端接口需严格校验权限。
  - 敏感操作（如删除文章）必须在前端实现二次确认弹窗。
- **数据处理**：
  - 管理端 Token 必须使用 `HttpOnly` Cookie 存储，防止脚本劫持。

### 3. 🎨 代码质量与风格
- **TypeScript 规范**：
  - 严禁滥用 `any` 类型，所有数据模型必须定义明确的 `interface`。
- **组件设计**：
  - 遵循单一职责原则，UI 与逻辑分离。
  - 优先复用 Shadcn UI 组件。

### 4. 🤝 协作与反馈机制
- **沟通原则**：
  - 在开发过程中，任何涉及架构变动或不确定的技术选型，必须及时暂停并与我确认。
- **文档维护**：
  - 关键逻辑必须编写 JSDoc 注释，新增功能同步更新 README。
